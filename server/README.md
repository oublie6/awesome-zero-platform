# Server

The repository's first runnable server is a minimal go-zero REST process under `apps/app-api`. It exposes health endpoints only and intentionally avoids database, cache, auth, metrics, tracing, deployment, or client scaffolding.

## Prerequisites

- Go `1.25.8`
- `goctl` `1.10.1`

Install `goctl`:

```bash
go install github.com/zeromicro/go-zero/tools/goctl@v1.10.1
```

## Structure

- `apps/app-api/app.api` — go-zero API contract source
- `apps/app-api/etc/main-api.yaml` — safe local development configuration
- `apps/app-api/internal/types` — generated request and response types
- `apps/app-api/internal/handler/routes.go` — generated route registration
- `apps/app-api/internal/logic` — generated logic scaffolds with handwritten health responses
- `apps/app-api/internal/bootstrap` — handwritten configuration loading and server startup

Generated files remain marked by goctl headers. Files marked `Code generated by goctl. DO NOT EDIT.` are regenerated from `app.api`. Files marked `Code scaffolded by goctl. Safe to edit.` are the intended handwritten extension points.

## Commands

From the repository root:

```bash
make generate
make fmt
make test
make run
```

The generation command run by `make generate` is:

```bash
cd server && goctl api go --api apps/app-api/app.api --dir apps/app-api --style gozero
```

If you prefer to work directly inside `server`:

```bash
go test ./...
find . -name '*.go' -type f -print0 | xargs -0 gofmt -w
go run ./apps/app-api -f apps/app-api/etc/main-api.yaml
```

## Startup

`make run` starts the API on the host and port configured in `apps/app-api/etc/main-api.yaml`.

The committed development configuration is:

```yaml
Name: main-api
Host: 0.0.0.0
Port: 8888
```

Startup fails with a non-zero exit if the config file is missing or if required settings such as `Name`, `Host`, or `Port` are invalid.

go-zero REST handles graceful shutdown by default. Sending `SIGINT` or `SIGTERM` stops the process cleanly.

## Health endpoints

Liveness:

```bash
curl -s http://127.0.0.1:8888/health/live
```

Expected response:

```json
{"status":"ok"}
```

Readiness:

```bash
curl -s http://127.0.0.1:8888/health/ready
```

Expected response:

```json
{"status":"ready"}
```
